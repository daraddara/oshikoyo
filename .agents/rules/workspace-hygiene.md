# プロジェクト直下へのファイル作成禁止ルール
エージェントは、作業中に生成する一時的なスクリプト、検証用の画像ファイル、コマンド実行時のログファイル（`> log.txt` など）を**絶対にプロジェクト直下（リポジトリルート）などの管理対象ディレクトリに配置してはいけません。**

## 一時ファイルの配置場所
- バックグラウンド実行のログファイルや検証スクリプトなどの一時ファイルを作成する場合は、必ずOSの一時ディレクトリ（`Linux/WSL` の場合は `/tmp/` フォルダ）を使用してください。
  - 〇 良い例: `npm run serve > /tmp/serve.log 2>&1 &`
  - × 悪い例: `npm run serve > serve.log 2>&1 &`
- 証拠として残す必要のない検証ファイルやダミーデータ等も同様に `/tmp/` を利用し、作業完了後は自動的にOSによって（または明示的に）破棄されるようにしてください。

## `.gitignore` への盲信禁止
「一時ファイルだからひとまず作成し、後で `.gitignore` に追加すればよい」という安易なルール追加を禁じます。`.gitignore` はあくまでプロジェクトのビルド成果物や既知のキャッシュ群を管理するものであり、エージェントがその場しのぎで作成したログファイル等で汚染しないでください。
