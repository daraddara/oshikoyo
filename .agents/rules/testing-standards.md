# テスト方針
- テストフレームワークは **Vitest**（または Jest）を使用してください。
- フォルダ構成: `tests/` ディレクトリ内に、対象ファイル名 + `.test.js` の形式で作成してください。

# JavaScriptのテスト
- カレンダーのロジック（日付計算、祝日判定など）は、境界値（月末、月初、閏年など）を重点的にテストしてください。
- DOM操作を伴う関数は、`jsdom` を使用してモックを作成し、イベントの発火と表示の変化を確認してください。

# CSS/UIのテスト
- クリティカルな表示要素（カレンダーのグリッドレイアウトなど）については、特定のクラスが付与されているか、または計算されたスタイル（computed style）が正しいかを確認するテストを記述してください。
