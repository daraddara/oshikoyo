# テストの基本原則
- **ロジックテスト**: `script.js` 内の計算（祝日、日付計算等）を変更した場合は、必ず Vitest でテストを実行し、100%パスすることを確認してください。
- **バグ修正時**: 修正を行う前に、まずそのバグを再現するテストケースを `test/` 内に追加してください。

# 記述スタイルと手法
- `script.js` は非モジュール（ESMではない）ため、テスト側では `fs.readFileSync` と `new Function` を使用し、特定のコメントブロック（例: `// --- Holiday Logic ---`）を基準にコードを抽出して検証してください。
- 新しい関数を追加する際は、テスト側で抽出しやすいように明確なコメント区切りを挿入してください。

# 検証項目
- 設定変更（週の開始日、表示月数）が正しく画面に反映されるかを確認してください。
- ダークモードやレスポンシブ表示において、視認性が損なわれていないか注意してください。

# 自律実行ルール (Autonomous Execution)
- `script.js` または `logic.test.js` を変更した後は、ユーザーからの指示を待たずに、自律的に `npm test` を実行して結果を確認してください。
- エージェントは今後、UIの確認を求められたら「ブラウザの直接操作」ではなく `npm run e2e` を実行し、生成されたレポートや画像ファイルから状態を判断してください。
- Jules 環境においても、UI に変更がある場合は自律的に `npm run e2e` を実行し、実行結果の画像を `tests/e2e/screenshots/` に保存してPRに反映してください。
- Playwright等を用いたE2Eの画像レイアウト変動および切り替えの検証には、必ず `tests/fixtures/images/` 配下の専用テスト画像を使用してください。
- もしテストが失敗した場合は、その原因を分析し、修正案を提示した上で再度テストを実行してください。
- 最終的に「テストがすべてパスした状態」で作業完了を報告してください。
