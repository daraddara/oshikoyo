# テストおよびサーバー実行ルール (Execute Tests & Server SOP)

このドキュメントは、プロジェクトにおけるサーバー起動およびテスト実行の標準手順を規定します。エージェントは自己判断で代替手段（Python等）を使用してはならず、必ず以下の手順を遵守してください。

## 1. 唯一の真実 (Single Source of Truth)
- サーバー起動およびテストに関するコマンドは、すべて `package.json` の `scripts` ブロックに定義されたものを使用してください。
- エージェントが独自の判断でプロジェクト外のツール（`python -m http.server` 等）を使用してサーバーを起動することを厳禁します。

## 2. サーバーの起動手順
- **標準コマンド**: `npm run serve` (内部的に `npx serve . -p 8081` を実行) を使用してください。
- **ポート**: 常に `8081` 番ポートを使用します。
- **事前確認**: 起動前に必ず対象ポートが解放されているか確認してください。
- **非ブロッキング起動 (Ready Check)**:
  - サーバーは終了しない永続的なプロセスであるため、`run_command` の `WaitMsBeforeAsync` を 1000〜2000ms 程度に設定し、直ちにバックグラウンドへ移行させてください。
  - **重要**: サーバーに対して `command_status` を呼び出し、プロセスの「終了」を待つことは**絶対に避けてください**。特に `WaitDurationSeconds` を大きな値（例: 300秒）に設定すると、サーバーが正常に稼働していてもワークフローが停止します。
  - サーバーの起動完了を確認するには、プロセスの終了を待つのではなく、`curl` コマンドや `read_url_content` ツールを使用して `http://localhost:8081` からレスポンスが返ってくるまで（最大数秒〜10秒程度）ポーリングを行ってください。
  - サーバーが一度「Serving!」と表示された後、`command_status` で状態を確認する場合は `WaitDurationSeconds: 0` を使用し、プロセスが「running」であることを確認するに留めてください。

## 3. 役割の分離 (Role Separation)

サーバーの起動とブラウザによるUI検証は、それぞれ独立したエージェントの責務とします。これらが混同されると、ポートの競合や接続エラー（ERR_CONNECTION_REFUSED）の原因となります。

- **メインエージェントの責務**: バックグラウンドでのサーバー起動、およびポートリスニングの完了確認。サーバーが完全に立ち上がり、URL (`http://localhost:8081`) が正常に応答するまで待機（ポーリングなど）する責任を持ちます。
  - **確認例 (Polling)**: `curl -s -o /dev/null -w "%{http_code}" http://localhost:8081/index.html` が `200` を返すまで数回試行する。
  - テスト後のクリーンアップも行います。
- **ブラウザサブエージェントの責務**: 原則として、自動化された E2E テスト（Playwright）による検証を優先してください。ブラウザサブエージェントによる手動操作は、自動テストの作成が困難な例外的なケースや、デバッグ目的のみに使用を限定します。サブエージェントが独自にサーバーを起動しようと試みることは**厳禁**です。

## 4. 環境クリーンアップ (OS別)
サーバー起動に失敗した場合やテスト終了時は、以下のコマンドでプロセスを完全に終了させてください。

### Windows (PowerShell)
- **プロセス終了**: `taskkill /F /IM node.exe /T` および `taskkill /F /IM chrome.exe /T` を使用（子プロセスを含めて強制終了）。
- **待機**: ファイルロック解除のため、終了後に 3〜5 秒の待機 (`Start-Sleep -s 5`) を推奨します。

### Linux / macOS (bash)
- **プロセス終了**: `npx kill-port 8081` および `pkill -9 chrome || true` を使用。
- **待機**: `sleep 5` を実行。

## 5. ブラウザ検証の要件
- ブラウザを起動する際は、必ず `.agents/rules/browser-isolation.md` の規定に従い、プロジェクトルートの隔離プロファイル (`.agents/browser_data`) を使用してください。
- URL は常にフルパス (`http://localhost:8081/index.html`) で指定し、相対パスでのアクセスを禁止します。

## 6. エラー時の対応
- `npm run serve` が失敗した場合、勝手に代替手段を探すのではなく、まず実行中の `node` プロセスを上記クリーンアップ手順で排除し、再試行してください。
