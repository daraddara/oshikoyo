
# /research ワークフロー実行に関するポリシー

このルールは、リポジトリの状態を調査し回答することに特化した `/research` ワークフローの運用方針を規定するものです。

## 1. 調査の徹底
- ユーザーの質問に対し、「おそらく〜だと思います」といった推測での回答を避け、必ず `view_file` や `grep_search` 等で実態を確認してください。
- 関連するソースコードだけでなく、`.agents/rules/` 内の関連ルール、`ISSUES.md` や `README.md` などのドキュメント、および最近のコミット履歴を横断的に調査してください。

## 2. ファイル変更の禁止 (Read-Only)
- このワークフローの実行中、指示のない限りプロジェクト本体のファイル（ソースコード、CSS、HTML、および既存の .md ドキュメント）を **`write_to_file` や `replace_file_content` 等で変更することを厳禁**します。
- `/research` の目的はあくまで「現状の把握と回答」であり、バグ修正や機能追加、ドキュメントの更新は別の適切なワークフロー（`/start`, `/feature`, `/sync` 等）で行ってください。

## 3. 回答の品質と構成
- 調査結果の回答は、専門用語を除き、すべて **日本語** で行ってください。
- 必要に応じて、調査対象のコードの抜粋やファイルパスを明示し、ユーザーが内容を容易に検証できるようにしてください。
- 複数の要因が絡む場合は、箇条書きやテーブルを用いて構造的に説明してください。

## 4. 提案とワークフロー移行
- 調査の結果、不具合の修正や機能の改善が必要であると判断した場合は、その内容を具体的に提案してください。
- ユーザーが提案に同意し、実装や修正に進む場合は、直ちにこの `/research` を終了し、`/start` または `/feature` 等の実装用ワークフローへ移行することを提案してください。

## 5. 例外事項
- 調査・分析のために一時的なスクリプトを `/tmp/` ディレクトリに作成して実行することは許可されます。
- `task.md` や `walkthrough.md` 等のエージェント管理用アーティファクトの更新は、通常通り行ってください。
