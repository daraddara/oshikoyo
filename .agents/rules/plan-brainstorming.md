# 目的
本格的な要件定義や実装に入る前に、ユーザーの「やりたいこと」を深掘りし、他のアプリの事例などを参考にしながら、実現可能性と方向性を探るためのブレインストーミングのルールです。

# ブレインストーミング時の指針
1. **ユーザー体験 (UX) の優先**:
   - 提案する機能は、ユーザーの「推し活」がより楽しく、便利になるかを第一に考えてください。
   - ただ機能を追加するだけでなく、UI/UXの観点から「どう見せるか」「どう操作させるか」も一緒に提案してください。

2. **多角的な調査**:
   - ユーザーから他の類似アプリ（推し活アプリやカレンダーアプリ等）についての言及があった場合や、UI/UXの参考が必要な場合は、自律的に `search_web` ツール等を用いてトレンドや機能事例を調査してください。
   - 調べた事例をそのまま取り入れるのではなく、このアプリ（おしこよ）のコンテキストや制約（Vanilla JS, LocalStorageなど）に合わせてどうアレンジできるかを提案してください。

3. **実現可能性のアセスメント**:
   - アイディアが現在の技術スタックで実現可能か、ざっくりとした評価を行ってください。
   - もし実装難易度が高い場合は、代替案や「まずはここまで作る」というスモールステップの提案も行ってください。

4. **モックアップやデザイン案の活用**:
   - 言葉だけでは伝わりにくいUIやレイアウトのアイディアは、HTML/CSSの簡易コードや、`generate_image` ツールを使ったイメージ画像などを作成して、視覚的に提案してください。
   - ただし、この段階では実際のプロジェクトのソースコード（.jsや.css等）の書き換えは行わず、あくまで「提案」の域に留めてください。

5. **柔軟な対話**:
   - ユーザーの最初のアイディアに対して「それは難しい」と即座に切り捨てるのではなく、「こうすればできるかもしれない」「こんなアプローチもある」と、建設的に議論を広げる姿勢を持ってみてください。
