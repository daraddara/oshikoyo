# ブランチクリーンアップ手順 (/cleanup)

このワークフローは、開発効率を維持するために、マージ済みまたは不要になった Git ブランチを整理する手順を規定します。

## 手順

### 1. リモートとの同期
最新のリモートの状態を反映し、すでに削除されたリモートブランチに対する追跡情報を削除します。
```bash
git fetch --prune origin
```

### 2. マージ済みブランチの特定
`main` ブランチにマージ済みのブランチをリストアップします。
- **ローカル**: `git branch --merged main`
- **リモート**: `git branch -r --merged origin/main`

### 3. 不要なブランチの削除
以下の基準に従ってブランチを削除してください。

#### ローカルブランチの削除
マージ済みのローカルブランチを一括削除する場合：
```bash
# 誤消去を防ぐため、一度リストを確認してから実行すること
git branch --merged main | grep -v "^\*" | grep -v "main" | xargs -r git branch -d
```

#### リモートブランチの削除
PRがマージまたはクローズされ、不要になったリモートブランチを削除します。
```bash
gh pr list --state merged --limit 100 --json headRefName -q ".[].headRefName" | xargs -I {} git push origin --delete {}
```
> [!TIP]
> `gh pr merge --delete-branch` を使用してマージしていれば、この操作は通常不要です。

### 4. クリーンアップ後の確認
整理が正しく行われたか確認します。
```bash
git branch -a
```

## 運用ルール
- 定期的（週に一度、または大きな機能リリースの後）にこのクリーンアップを実行してください。
- 作業中のブランチや、意図的に残している実験用ブランチを誤って消さないよう注意してください。
