# 手順
1. **コンテキストの同期**:
   - `ISSUES.md` を読み込み、現在の未完了タスクとプロジェクトの優先順位を確認してください。
   - `design-standards.md` を読み込み、UI/UXの基本方針（ダークモード対応、ガラスモーフィズム等）を再確認してください。

2. **多角的な分析（ブレインストーミング）**:
   - ユーザーのアイディアに対し、以下の3つの視点でフィードバックを行ってください：
     - **ユーザー体験 (UX)**: 「推し活」がより楽しく、便利になるか？
     - **実装の容易性**: `localStorage` や既存の祝日ロジックと競合しないか？
     - **保守性**: 将来的な機能拡張（例：複数カレンダー管理など）を阻害しないか？

3. **技術的な具体案の提示**:
   - 相談内容が具体的な機能であれば、擬似コードやデータ構造（JSON案など）を提示して、実装のイメージを共有してください。
   - ただし、この段階で**実際のソースコード（.jsや.css）を編集してはいけません。**

4. **合意形成とドキュメント化**:
   - ディスカッションの結果、実装に進むことが決まった場合は、以下の作業を提案または実行してください：
     - `ISSUES.md` に新しい Issue ID（A-09など）として要件を追記する。
     - 複雑な仕様の場合は、`docs/design/` 配下に設計メモを作成する。

5. **ネクストステップの提示**:
   - 最後に、このアイディアを `/start` するために必要な準備（追加調査や素材の用意など）を日本語で簡潔にまとめてください。
