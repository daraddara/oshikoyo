# 手順
1. **対象の限定**: 
   - 変更が `.md`, `.json`, `.txt` などの非コード資産のみであることを確認してください。
   - 万が一ソースコード（.js, .css, .html）が含まれる場合は、このワークフローを中止し `/start` を促してください。
2. **最新の取得**: `main` ブランチにいることを確認し、`git pull` を実行して最新状態にしてください。
3. **ドキュメントの反映**: ユーザーが指示したドキュメント（例: `ISSUES.md`）の更新、または `plan.md` からの転記を実行してください。
4. **即時コミット**:
   - ブランチは作成せず、直接 `main` ブランチでコミットしてください。
   - コミットメッセージは `docs: ` または `chore: ` を接頭辞として使用し、日本語で概要を記述してください。
   - 例: `docs: ISSUES.md に A-09 (設定画面UX改善) を追加`
5. **完了報告**: 「ドキュメントの更新を main に直接反映しました」と日本語で報告してください。
