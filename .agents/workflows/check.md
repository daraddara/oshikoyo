# 手順
1. **サーバー起動とポートの厳格な確認 (メインエージェントの義務)**:
   - ターミナルで `npx serve . -p 8081` を実行してください。
   - **重要（起動確認）**: 起動直後に必ず `netstat -ano | findstr :8081` を実行し、状態が `LISTENING` であることを目視確認してください。
   - **競合発生時の対応**: もし出力（INFO）で 8081 以外のポート番号が表示されたり、`netstat` で確認できなかった場合は、以下のコマンドで古いプロセスを掃除してから再試行してください：
     `Get-Process -Name node -ErrorAction SilentlyContinue | Stop-Process -Force`

2. **ブラウザへの指示 (ブラウザエージェントの義務)**:
   - **必ず `http://localhost:8081` を開いて検証を開始してください。** セキュリティポリシーにより `file:///` 形式の直接アクセスは禁止されています。
   - 接続に失敗した場合は無限にリトライせず、即座にメインエージェントに制御を戻してサーバーの状態を確認させてください。
   - ページを開いた直後に、ブラウザコンソールで `localStorage.clear()` を実行し、リロードして「新規導入時の状態」から検証を開始してください。

3. **ビジュアル確認**:
   - ダークモードとライトモードを切り替え、すべてのテキストが視認可能か確認してください。
   - 画面幅を変更し、レスポンシブ表示（特にカレンダーのグリッド）が崩れていないか確認してください。

4. **機能動作確認**:
   - 設定パネルを開き、週の開始日や表示月数を変更して「保存」が反映されるか確認してください。
   - カレンダーの「前月/次月」ボタンが正しく動作するか確認してください。

5. **推し設定確認**:
   - 推しの追加、削除、インポート、エクスポートが機能することを確認してください。
   - 設定した推しの情報がカレンダーに反映されること、および推しカラーの視認性をチェックしてください。

6. **課題照合**:
   - `ISSUES.md` に記載されている未完了の懸念点（例：モバイル表示 A-04）が改善されているか確認してください。

7. **検証と記録**:
   - 検証項目、スクリーンショット、動作確認動画（Playback）を含めた結果を日本語で報告してください。
   - スクリーンショットは必ずプロジェクトルートの `screenshots/` フォルダに保存し、ファイル名に Issue ID を含めてください（例: `A-09-result.png`）。

8. **報告書の作成 (Walkthrough)**:
   - 画像埋め込みには `./screenshots/A-09-result.png` のような**相対パス**を使用してください。
   - **パスにドライブレター（C:等）やOSの絶対パスが含まれていないか、厳重に確認してください。** 絶対パスが含まれるとバリデーションエラーになります。

9. **クリーンアップ（自動強制終了）**:
   - 報告完了後、直ちにブラウザウィンドウを閉じてください。
   - **ポート8081を使用しているサーバープロセスを、`Stop-Process` や `taskkill` を用いて強制終了させてください。** 次回の競合を防ぐため、確実に応答を断ち切ってください。
