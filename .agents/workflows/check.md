# 手順
1. **サーバー起動**: ローカルサーバー（`npx serve .` 等）を起動し、ブラウザでアプリを開いてください。
2. **ブラウザ初期化**: 
   - ページを開いた直後に、ブラウザのコンソールまたはスクリプト実行機能を用いて `localStorage.clear()` を実行し、キャッシュや以前の設定を完全に消去してください。
   - ページをリロードし、デフォルト設定（新規導入時の状態）でカレンダーが表示されることを確認してから検証を開始してください。
3. **ビジュアル確認**:
   - ダークモードとライトモードを切り替え、すべてのテキストが視認可能か確認してください。
   - 画面幅を変更し、レスポンシブ表示（特にカレンダーのグリッド）が崩れていないか確認してください。
4. **機能動作確認**:
   - 設定パネルを開き、週の開始日や表示月数を変更して「保存」が反映されるか確認してください。
   - カレンダーの「前月/次月」ボタンが正しく動作するか確認してください。
5. **推し設定確認**:
   - 設定パネルを開き、推しの追加、削除、インポート、エクスポートが機能することを確認してください。
   - 設定した推しの情報がカレンダーに表示反映されることを確認してください。
   - いくつかの推しカラーでの表示でカレンダー上、視認しにくい色の組み合わせにならないかチェックしてください。
6. **課題照合**: `ISSUES.md` に記載されている「モバイル表示の最適化 (A-04)」などの懸念点が解消されているか確認してください。
7. **報告**: 確認した項目と、スクリーンショットに基づいた結果を日本語で報告してください。
8. **クリーンアップ（自動終了）**:
   - **報告が完了し、ユーザーへの提示が終わった直後に、起動したブラウザウィンドウを必ず閉じてください。**
   - **動作確認に使用したサーバープロセス（`npx serve` 等）もあわせて終了させてください。**
   - **サーバープロセスを taskkill や Stop-Process を使って強制終了させ、応答を待たずに報告を完了してください **
