## 2024-05-15 - DOM Batching and Perceived Performance
**Learning:** Using `DocumentFragment` is highly effective for batching synchronous DOM insertions to prevent layout thrashing and improve rendering performance. However, applying this optimization inside asynchronous loops (e.g., `for...of` with `await` for image loading) blocks progressive rendering. The user must wait for *all* asynchronous operations to finish before seeing *any* elements, significantly degrading the perceived performance of the UI.
**Action:** When rendering lists or components dynamically, only use `DocumentFragment` for batching synchronous DOM operations. For asynchronous lists (like fetching or processing images one by one), append elements directly to the live DOM sequentially so they appear progressively to the user as they load.
