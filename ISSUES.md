# 課題管理 (Issue Management)

プロジェクトの動作仕様が曖昧な点や、今後検討が必要な項目をリスト化しています。技術的な決定事項や運用ルールについては、`.agent/rules/` 内のドキュメントが唯一の真実（SSOT）となります。

## 動作仕様の曖昧な箇所

| ID | 項目 | 内容 | ステータス |
|:---|:---|:---|:---|
| A-01 | 祝日ロジックの保守 | 現在の祝日計算はコード内にハードコードされています。将来的な法律改正（例：山の日やスポーツの日の移動）にどう対応するか。 | 完了 |
| A-02 | 12ヶ月以上の表示 | 現在は1〜3ヶ月を想定していますが、4ヶ月以上表示した場合のレイアウト（スクロール量やパフォーマンス）の挙動。 | 検討中 |
| A-03 | ブラウザストレージの永続性 | `localStorage`がクリアされた際、デフォルト値に戻るが、その際ユーザーに通知するかどうか。 | 検討中 |
| A-04 | モバイル表示の最適化 | デスクトップPWAを主眼においていますが、極端に狭い画面（スマホ等）での縦スクロール/横スクロールの優先順位。 | 検討中 |
| A-05 | 過去・未来の境界 | 何年前まで、または何年先までカレンダーを遡れる/進められるか。 | 未着手 |
| A-06 | ダークモード切替UI | `design-standards.md` にダークモード対応の規定がありますが、現在UI上に切り替えスイッチが実装されていません。 | 未着手 |
| A-07 | 祝日データの外部API取得 | 将来的な法改正に対し、ユーザー自身が外部APIから最新の祝日データを取得しアップデートできるオプション機能の提供。 | 未着手 |
| A-08 | デフォルト画像の非表示問題 | デフォルトでの画像表示がされない（初期表示されない）問題。原因確認と修正が必要。 | 完了 |
| A-09 | 設定画面のUX改善 | 直接操作が増えたことで設定画面の項目が重複。項目の整理とリセット機能の検討。 | 検討中 |

## 改善要望・機能追加案

- [x] カスタム休日の設定機能（「推し」の誕生日や記念日を赤く塗るなど）
    - **要望**: 「推し」の情報管理IF（名前、誕生日、デビュー日、カラーコード等）を含めて実装。
- [x] 任意のコンテンツ表示領域の追加
    - **要望**: 推しの画像やファンアートを表示。URL指定、リストからのランダム表示、サイクル表示などの機能を備える。
- [x] 画像・カレンダー配置の直感的なUI操作
    - **要望**: 設定画面を開かず、ドラッグ＆ドロップやスプリッターで直感的に配置・サイズ変更を行えるようにする。
- [x] 複数月表示時の縦横レイアウトの直感操作
    - **要望**: メイン表示画面上のボタンからダイレクトにカレンダーの並びを縦・横に切り替えられるようにする。
- [ ] 印刷用CSSの最適化
- [ ] 設定画面のUX改善と役割の整理 (A-09)
    - **要望**: 直接操作（ドラッグ、ヘッダーボタン）可能な項目を設定画面から削除し、画面をシンプルにする。
    - **内容**: 表示月数、レイアウト方向、画像位置などの重複を排除し、「配置をデフォルトに戻す」ボタンや操作案内のテキストを追加する。
