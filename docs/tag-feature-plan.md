# 推し・画像連携タグ機能 要件と実装計画

推しと画像に横断的に「タグ」を紐付け、整理・検索・優先表示など柔軟な連携体験を提供する「タグ機能」の実装計画です。洗い出した多様なユースケースをもとに、段階的な実装への土台を構築します。

## ユースケースの洗い出し（アイデアレベル）
ユーザー様の事前アイデアを元に、タグ機能の可能性を整理・拡張しました。

### A. 画像と推しの連携・優先表示
- **記念日優先表示**: 推しの名前や記念日名がタグ付けされた画像を、推しの誕生日等のイベント日に優先ピックアップしてメイン画面に表示する。
- **ユニット/グループ連携**: 推しAと推しBの両方に「ユニットX」というタグを付け、画像にも「ユニットX」タグを付けることで、画像に複数の推しが写っている場合の緩やかな関連付けを容易にする。

### B. 整理・検索・絞り込み
- **一覧でのスマートフィルター**: 画像一覧画面で「絵師名」「イベント名」「年」で絞り込み、目当ての画像に瞬時にアクセスできるようにする。
- **エクスポート制御**: 特定のタグ（例：「エクスポート除外」「公開用」）を利用して、データエクスポート時の出力対象を細かくコントロールする。
- **並び順のグルーピング**: 画像一覧において、登録順とは別に特定のタグ順でグルーピング表示を行う。

### C. シームレスなタグ付与UX
- **登録時の即座・一括付与**: 画像をドロップ（あるいはペースト/ファイル選択）した瞬間に「プレビュー確認・タグ付与機能をもつダイアログ」を表示し、シームレスにタグを入力してから保存する。
- **インライン・クイック編集**: 表示状態（ライトボックス）や一覧状態から、スムーズにタグを追加・削除できるUX（オートコンプリートによる入力補完機能付き）。

## データ構造の採用候補（土台案）
パフォーマンスを担保しつつ、**複雑化を避けるため極めてシンプル**な文字列ベースの構造を採用します（階層化や色メタデータ等の過剰な複雑化は行いません）。

- **`appSettings.tags` (新規追加)**
  - 既存の全タグを管理し、サジェスト（入力補完）や一覧抽出に高速にアクセスするためのマスターリスト。（純粋な文字列配列 `string[]`）
- **`appSettings.oshis[].tags` (新規追加)**
  - 各「推し」に紐づくタグリスト。
- **画像メタデータ内の `tags` (新規追加)**
  - IndexedDB等で管理される各画像レコードに `tags: string[]` を追加拡張。

## 段階的な実装プラン
一度に全機能を実装すると複雑化・バグのリスクが高まるため、以下のフェーズに分けて安全に導入します。

### Phase 1: データ基盤とタグ管理の基本UI
- 設定画面（推しデータ）にタグ入力UIを追加。
- 画像一覧画面、および個別表示（ライトボックス）において、各画像へのタグ閲覧・追加・削除ができるUIの実装。
- 入力時の既存タグ・サジェスト（入力補完）機能のベース作成。

### Phase 2: 画像登録時のプレビュー付与機能
- 画像のD&D・ペースト・ファイル選択によるインポート時に「インポート時プレビュー・タグ付与ダイアログ」を介すフローを構築。複数画像の同時ドロップ時のUX（一斉付与など）も考慮。

### Phase 3: フィルタ・連携ロジックの実装
- 画像一覧画面へのタグフィルター（絞り込み・検索）機能の追加。
- メイン画面（カレンダー等）において、記念日発生時にタグを参照した優先表示アルゴリズムへ改修。
- データエクスポート時のフィルタリング機能への応用。
