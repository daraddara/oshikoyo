# 課題管理 (Issue Management)

プロジェクトの動作仕様が曖昧な点や、今後検討が必要な項目をリスト化しています。技術的な決定事項や運用ルールについては、`.agents/rules/` 内のドキュメントが唯一の真実（SSOT）となります。

## 動作仕様の曖昧な箇所

| ID | 項目 | 内容 | ステータス |
|:---|:---|:---|:---|
| A-01 | 祝日ロジックの保守 | 現在の祝日計算はコード内にハードコードされています。将来的な法律改正（例：山の日やスポーツの日の移動）にどう対応するか。 | 完了 |
| A-02 | 12ヶ月以上の表示 | レイアウト崩れとユーザビリティの観点から最大2ヶ月に制限。3ヶ月以上の表示は非対応として整理。 | 完了 |
| A-03 | ブラウザストレージの永続性 | `localStorage`がクリアされた際、デフォルト値に戻るが、その際ユーザーに通知するかどうか。 | 検討中 |
| A-04 | モバイル表示の最適化 | デスクトップPWAを主眼においていますが、極端に狭い画面（スマホ等）での縦スクロール/横スクロールの優先順位。 | 検討中 |
| A-05 | 過去・未来の境界 | 何年前まで、または何年先までカレンダーを遡れる/進められるか。 | 未着手 |
| A-06 | ダークモード切替UI | `design-standards.md` にダークモード対応の規定がありますが、現在UI上に切り替えスイッチが実装されていません。 | 未着手 |
| A-07 | 祝日データの外部API取得 | 将来的な法改正に対し、ユーザー自身が外部APIから最新の祝日データを取得しアップデートできるオプション機能の提供。 | 未着手 |
| A-08 | デフォルト画像の非表示問題 | デフォルトでの画像表示がされない（初期表示されない）問題。原因確認と修正が必要。 | 完了 |
| A-09 | 設定画面のUX改善 | 直接操作が増えたことで設定画面の項目が重複。項目の整理とリセット機能の検討。 | 完了 |
| A-10 | カレンダー縦並び時の画像幅 | カレンダーを縦に並べた際、画像エリアが表示月数に合わせて横に伸びてしまう不具合。 | 完了 |
| A-11 | 通常UIからの画像表示モード切り替え | 設定画面を経由せず、メイン画面の画像エリアから直接表示モード（固定、ランダム等）を切り替えられるようにする。非表示オプションの廃止とアイコンUI化完了。 | 完了 |
| A-12 | テスト用画像の拡充と検証強化 | 各種アスペクト比（16:9, 9:16, 1:1など）の画像をFixturesに導入し、画像表示関連のUI崩れやレイアウトのテスト検証を詳細化。 | 完了 |
| A-13 | ユニットテストのモック堅牢性向上 | `tests/security.test.js` 等で `script.js` のコードを直接抽出（extractCode）し、JSDOMでモックしている仕組みが脆弱なため、ロジック変更やレイアウト関数の追加でテストが壊れやすい問題を改善する。 | 未着手 |
| A-14 | 全データバックアップ | 全設定と画像をGzip圧縮して1ファイルでエクスポート・インポートする機能の不足。 | 完了 |
| A-15 | ブラウザストレージの容量管理 | 高画質画像の多数登録による容量圧迫への対策（自動圧縮・一括圧縮）。 | 完了 |
| A-16 | 画像タグパッケージのエクスポート | 特定の画像とそれに関連するタグ情報をパッケージ化して配布・共有する機能。 | 完了 |
| A-17 | 推し情報のCSVインポート | 他のツールや手書きCSVのリソースから、名前や誕生日などの情報を一括で登録する機能。 | 完了 |
| A-18 | 画像・推しのタグ管理 | 複数の画像や「推し」に対して、属性を示すタグを付与・検索・分類する基盤。 | 完了 |

## 改善要望・機能追加案

- [x] カスタム休日の設定機能（「推し」の誕生日や記念日を赤く塗るなど）
    - **要望**: 「推し」の情報管理IF（名前、誕生日、デビュー日、カラーコード等）を含めて実装。
- [x] 任意のコンテンツ表示領域の追加
    - **要望**: 推しの画像やファンアートを表示。URL指定、リストからのランダム表示、サイクル表示などの機能を備える。
- [ ] 画像の魅力を高める「背景ぼかし（シアターモード）」の導入
    - **要望**: 画面左右などの余白に、同じ画像を拡大して強くぼかしたものを背景として表示し、没入感を高める。
- [ ] カレンダー全体を透過させる「全画面壁紙モード」の追加
    - **要望**: 画像を画面全体の背景（Wallpaper）として固定表示し、ガラスモーフィズムのUI越しに透けて見えるようにする。
- [x] 画像・カレンダー配置の直感的なUI操作
    - **要望**: 設定画面を開かず、ドラッグ＆ドロップやスプリッターで直感的に配置・サイズ変更を行えるようにする。
- [x] 複数月表示時の縦横レイアウトの直感操作
    - **要望**: メイン表示画面上のボタンからダイレクトにカレンダーの並びを縦・横に切り替えられるようにする。
- [x] 画像切り替え間隔のプリセット化と特定時刻更新
    - **要望**: 設定画面の複雑なインターバル設定を廃止し、画像エリアで直感的に切り替えられるプリセットUIを導入。毎日0時/4時などの特定時刻更新にも対応。
- [ ] 印刷用CSSの最適化
- [x] 設定画面のUX改善と役割の整理 (A-09)
    - **要望**: 直接操作（ドラッグ、ヘッダーボタン）可能な項目を設定画面から削除し、画面をシンプルにする。
    - **内容**: 表示月数、レイアウト方向、画像位置などの重複を排除し、「配置をデフォルトに戻す」ボタンや操作案内のテキストを追加する。
- [x] スマホからの画像登録UX改善（Web Share Target API & クリップボードAPI）
    - **要望**: スマホブラウザからの画像登録を手軽にするため、Android向けにOSの共有メニューから直接アプリに送れる機能（Web Share Target API）と、iOS/他環境向けにクリップボードから画像を直接貼り付けられる機能のハイブリッド実装を行う。
- [x] 画像・推しへのタグ付け機能
    - **要望**: 画像や推しに対して複数のタグを設定し、管理・分類を行えるようにする。
- [x] 全データバックアップとストレージ最適化 (A-14, A-15, A-16)
    - **要望**: ブラウザストレージの容量制限に対応するための画像圧縮機能と、他デバイスへの移行を目的としたGzip圧縮バックアップ機能を実装する。
- [x] CSV形式での「推し」情報一括登録 (A-17)
    - **要望**: 既存の推し情報（名前、記念日など）をCSVファイルから一括で読み込み、登録・更新を容易にする。
