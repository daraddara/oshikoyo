# 実施計画：ライセンス表記の整備とアプリケーションライセンスの導入

## 1. 目的と背景
- **目的**: 使用している外部リソース（フォント、アイコン、API）のライセンス要件を満たし、開発コミュニティへの謝辞を適切に表示する。また、本アプリ自体の利用許諾範囲（MITライセンスを想定）を明確にする。
- **背景**: 調査の結果、現在「アプリ情報」パネルにはプライバシーポリシー等は記載されていますが、使用している各ライブラリの具体的なライセンス情報が欠落していることが判明しました。

## 2. 実装内容の概要

### A. プロジェクトライセンスの追加
- プロジェクトルートに `LICENSE` ファイル（MIT License）を新規作成。
- `package.json` に `"license": "MIT"` を追記。
- `README.md` の末尾にライセンスに関する章を追加。

### B. UIへの「サードパーティライセンス」項目の追加
設定画面（デスクトップ・モバイル両方）の「アプリ情報」パネルに、以下の情報をまとめたセクションを追加します。

| 項目 | 内容 | 表記先URL |
| :--- | :--- | :--- |
| **Fonts** | M PLUS Rounded 1c, Noto Sans JP, Outfit (SIL Open Font License 1.1) | https://scripts.sil.org/OFL |
| **Icons** | Lucide Icons (ISC License) | https://lucide.dev/license |
| **API** | Holidays JP API | https://holidays-jp.github.io/ |

## 3. 具体的な修正予定箇所

### 1. `LICENSE` (新規作成)
標準的な MIT ライセンスのテキストを配置。

### 2. `index.html`
- モバイル用サブパネル (`mobileSubPanel-appinfo`) とデスクトップ用タブパネル (`tabPanelAppInfo`) の両方に、ライセンス情報を表示するための HTML 構造を追加。

### 3. `src/style.css`
- ライセンス一覧を表示するためのスタイル（リンクの装飾やリストの余白等）を追記。

### 4. `README.md`
- ライセンスの項目を追加し、MITライセンスであることを明記。

## 4. 期待される効果
- **法的コンプライアンスの向上**: 各リソースのライセンス条件を遵守できます。
- **信頼性の向上**: アプリ情報の充実により、ユーザーが安心して利用できる「丁寧な作り」を印象付けられます。
- **プロジェクトの明文化**: 第三者がコードを利用・貢献する際のルールが明確になります。
