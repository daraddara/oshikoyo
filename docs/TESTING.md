# テスト実施ルールとガイドライン (Testing Guide)

本プロジェクトの品質を維持するためのテスト実施ルールを定義します。

## テストの種類

### 1. ロジックテスト (Unit Testing)
- **対象**: `script.js` 内の計算ロジック（祝日計算、日付計算）。
- **ツール**: Vitest
- **必須事項**: 計算ロジックに変更を加えた場合は必ず実行し、100%パスすること。

### 2. 機能・設定テスト (Functional Testing)
- **対象**: 設定変更（週の開始日、表示月数、レイアウト）の反映。
- **チェック項目**:
  - 設定変更後、「保存」で正しく画面が更新されるか。
  - リロード後も設定が維持されているか。
  - カレンダーのナビゲーション（前/次）が正しく動作するか。
  - 日付クリックで「今日」に戻るか。

### 3. 表示テスト (UI Testing)
- **対象**: レスポンシブ、ダークモード。
- **チェック項目**:
  - ダークモード切り替え時に文字が読めるか。
  - 画面幅を縮めてもレイアウトが崩れず、内容が確認できるか。

### 4. 画像のレイアウトと切り替えテスト
- **対象**: アスペクト比の異なる画像設定時のレイアウト維持、画像切り替えの動作確認。
- **使用するFixture**: `tests/fixtures/images/` 配下のテスト画像（横長: `test_landscape.png`, 縦長: `test_portrait.png`, 正方形: `test_square.png`）など。
- **チェック項目**:
  - カレンダーの縦・横レイアウトに応じ、画像エリアが期待通りにリサイズ、もしくはクロップ（`object-fit`等）されているか。
  - 画像変更時にエラーなく表示が切り替わるか。

---

## 安定したテストのための指針 (Robustness)
異なる実行環境（WSL, CI等）で安定して動作させるため、以下の点に留意してください：
- **状態待ち**: `page.waitForTimeout()` を避け、要素の出現や変化を待つロジックを使用する。
- **日付の固定**: スクリーンショットの不整合を防ぐため、基準日（2024-01-01）をモックする。
- **タイムアウト調整**: 必要に応じて `TEST_TIMEOUT_FACTOR` を使用する。
