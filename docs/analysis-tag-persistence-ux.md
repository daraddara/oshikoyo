# 画像登録時のタグ引き継ぎ仕様に関する調査・改善提案

## 1. 現状の仕様
現在、おしこよの画像登録フロー（プレビュー画面）では以下の挙動が実装されています。

- **タグの自動補完**: 画像を選択してプレビュー画面が開く際、「前回最後に登録完了した画像」に付与されていたタグが自動的に入力欄にセットされる。
- **永続化**: この履歴はブラウザの `localStorage` (`oshikoyo_state`) に保存されており、セッションを跨いでも保持される。

## 2. 背景と設計意図
この仕様は、以下の背景に基づいて実装されました。

- **一括登録の省力化**: イベントやライブの画像など、同じ属性（タグ）を持つ大量の画像を連続して登録する際、毎回同じタグを入力する手間を省くため。
- **「共通タグ」の概念**: 全画像に一括して同じタグを付ける「共通タグ」機能を設けることで、登録後の編集コストを下げることを目的としていた。

## 3. 顕在化した課題
実際の利用シーンにおいて、以下の問題点が指摘されています。

### A. 意図しないタグの付与（タグ汚染）
異なる属性の画像を登録する際、前回のタグが残っていることに気づかず「追加」ボタンを押してしまうと、過去のタグが不要に付与されてしまう。これを修正するには登録後に一枚ずつ編集する必要があり、かえって手間が増える。

### B. UIの文脈不一致
現在のラベルは「共通タグ（全画像に付与・任意）」となっているが、**単一のファイルを登録する場合**においても「共通」という言葉が表示される。これはユーザーにとって「何と共通なのか？」という混乱を招く原因となっている。

### C. 挙動の不透明性
ユーザーが特定の操作をしていないのに、入力欄に値が既に入っているという状態は、初見のユーザーにとって予期しない（マジックな）挙動に見え、不親切に感じられる可能性がある。

## 4. 改善アイディアの提案

これらの課題を解決するため、安全性を高めつつ利便性を維持する以下の変更案を提案します。

### ① デフォルト「空」への変更と履歴チップの導入
- **変更点**: プレビュー画面の初期状態は常に「空」とする。
- **利便性ケア**: 入力欄の下に「前回のタグ: [タグ名×]」といったチップを表示。クリックすることで、ワンタッチで前回のタグを流用可能にする。

### ② 文言の簡略化と動的変更
- **ラベルの変更**: 基本的なラベルをシンプルに **「タグ」** に統一する。
- **動的な補足**: 登録枚数が2枚以上のときのみ、「すべての画像に付与されます」といった補助説明を表示する、またはプレースホルダーを活用する。

### ③ 引き継ぎ設定のオプション化
- **構成**: 設定画面に「前回登録時のタグを引き継ぐ」というトグルスイッチを追加。
- **効果**: 自分のワークフローに合わせて、自動入力か手動入力をユーザーが選択できるようにする。

---

## 5. 結論と推奨アクション
「共通タグ」という言葉による違和感を解消し、誤登録リスクを最小限にするため、**「ラベルを『タグ』に簡略化」**しつつ、**「初期状態を空にする（任意で前回分を復元できるインターフェースを設ける）」**方向での改修が望ましいと考えられます。
