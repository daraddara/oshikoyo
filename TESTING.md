# テスト実施ルールとガイドライン (Testing Guide)

本プロジェクトの品質を維持するためのテスト実施ルールを定義します。

## テストの種類

### 1. ロジックテスト (Unit Testing)
- **対象**: `script.js` 内の計算ロジック（祝日計算、日付計算）。
- **ツール**: Vitest
- **実行方法**: `npm test`
- **ルール**: 計算ロジックに変更を加えた場合は必ず実行し、100%パスすることを確認すること。

### 2. 機能・設定テスト (Functional Testing)
- **対象**: 設定変更（週の開始日、表示月数、レイアウト）の反映。
- **ツール**: ブラウザによる手動検証 or Playwright（将来導入）
- **チェック項目**:
  - [ ] 設定変更後、「保存」で正しく画面が更新されるか。
  - [ ] リロード後も設定が維持されているか。
  - [ ] カレンダーのナビゲーション（前/次）が正しく動作するか。
  - [ ] 日付クリックで「今日」に戻るか。

### 3. 表示テスト (UI Testing)
- **対象**: レスポンシブ、ダークモード。
- **チェック項目**:
  - [ ] ダークモード切り替え時に文字が読めるか。
  - [ ] 画面幅を縮めても横スクロールで内容が確認できるか。

## 環境構築

開発環境で自動テストを実行するには、以下のセットアップが必要です。

1. Node.js がインストールされていることを確認。
2. リポジトリルートで `npm install` を実行。
3. `npm test` でテストを実行。

## テスト追加時のルール

1. **バグ修正時**: バグを再現するテストケースを `tests/` 内に追加してから修正すること。
2. **新機能追加時**: 期待される動作（仕様）に基づいたテストを追加すること。
